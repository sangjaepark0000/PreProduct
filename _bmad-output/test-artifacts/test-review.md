## Findings Applied

### HIGH: DB-backed E2Es failed inside the app server when `DATABASE_URL` was absent

- **Evidence:** Full `pnpm test:ci` reached DB-save dependent Playwright tests without `DATABASE_URL`, causing `DATABASE_URL is required to use the Prisma listing repository.` from the spawned Next.js server.
- **Fix:** `playwright.config.ts` now forwards the dotenv-loaded parent environment to the web server process. DB-backed listing E2Es now declare an explicit `test.skip` prerequisite when `DATABASE_URL` is missing, and `tests/README.md` documents the behavior.
- **Additional scope:** The older red-phase Story 1.2 DB E2E also submits the listing form, so it received the same skip guard to prevent full `test:ci` from failing on the same environment gap.

### MEDIUM: DB E2Es used wall-clock suffixes without needing runtime uniqueness

- **Evidence:** `tests/e2e/listing-registration.spec.ts` and the Story 1.2 red-phase status spec used `Date.now()` only to create display titles.
- **Fix:** Replaced those suffixes with stable titles. Listing IDs are generated by the application, and duplicate titles are valid, so runtime time was unnecessary.
