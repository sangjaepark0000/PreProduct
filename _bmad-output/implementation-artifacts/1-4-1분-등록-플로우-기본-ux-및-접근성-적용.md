I want 모바일 기준의 단순 단계형 등록 플로우를 사용하고 싶다,

**Scope Tag:** Active MVP
**Story Type:** Feature
**FR Trace:** FR1, FR4, FR5, FR7

## Dev Notes

### Epic Context

- Epic 1의 마지막 스토리로서, Story 1.1~1.3에서 구현된 등록/조회/수정/복구 기능에 모바일 단계 UX와 접근성 하한선을 추가한다.
- Story 2.x(사진 업로드/AI 검토/추천가 확정)의 컴포넌트 축을 선행 정렬해 이후 구현 시 파일 위치/역할 충돌을 방지한다.

### Previous Story Intelligence (Story 1.3)

- Story 1.3에서 `ListingDraftForm`은 필드 오류/복구 가이드/동일 화면 재시도 경로가 고정되었다. Story 1.4는 이 복구 동작을 변경하지 않는다.
- 재시도는 마지막 요청 액션(`create`/`update`/`reload`)과 일치해야 하며, 입력값 손실은 금지된다.
- 기존 회귀 금지 포인트: stale draft 정리, keySpecs 줄바꿈 파싱, 실패 후 상태 초기화 규칙.

### Current Codebase Intelligence

- 엔트리 경로는 `preproduct/src/app/(app)/page.tsx`이며 현재 `StatusDiagnosticForm` + `ListingDraftForm` 조합으로 구성된다.
- `ListingDraftForm`은 API 계약(`{ data, meta }` / `{ error }`)과 필드별 오류 표시를 이미 사용한다.
- 현재 스타일은 `entry-page.module.css`, `listing-draft-form.module.css` 중심이며 44x44/포커스 기준을 전역으로 강제하는 구조는 미흡하다.

### Architecture Compliance (Must Follow)

- 레이어 경계: `feature -> domain -> infra`만 허용, `feature -> infra` 직접 import 금지
- 매핑 경계: casing 변환은 `src/infra/mapper` 전용
- API 응답 형식: 성공 `{ data, meta }`, 실패 `{ error: { code, message, details?, requestId } }`
- Scope 우선순위: Active MVP > Deferred P1.5+ > Legacy Reference
- 변경 순서: `UI contracts/state -> components -> styles -> tests`

### Technical Requirements

- 모바일 기준 화면폭 320-767에서 등록 흐름 핵심 정보(현재 단계/남은 단계/다음 행동)가 초기 가시영역에 노출되어야 한다.
- 주요 상호작용 컨트롤(버튼, 입력, 선택)은 최소 44x44 터치 타깃을 만족해야 한다.
- 키보드 전용 사용자도 등록 시작 -> 저장/수정 -> 재시도 흐름을 완료할 수 있어야 한다.
- 상태 전달은 색상 외 텍스트/아이콘을 병행해야 하며, 오류 복구 가이드가 항상 유지되어야 한다.
