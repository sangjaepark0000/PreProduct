# Story 1.2: 프리리스팅 조회 및 수정

Status: done

## Story

As a 판매 사용자,  
I want 저장한 프리리스팅을 조회하고 수정하고 싶다,  
so that 등록 상태를 반복적으로 업데이트할 수 있다.

**Scope Tag:** Active MVP  
**Story Type:** Feature  
**FR Trace:** FR1, FR2, FR3, FR4, FR5, FR6, FR7, FR16

## Acceptance Criteria

1. **Given** 본인이 생성한 프리리스팅이 있을 때  
   **When** 상세 화면에서 항목을 수정 후 저장하면  
   **Then** 변경 내용이 반영된다  
   **And** 수정 시각이 최신으로 갱신된다.

## Dev Notes

### Epic Context

- Story 1.2는 Story 1.1에서 만든 draft 생성 경계 위에 조회/수정 경계를 추가하는 스토리다.
- 본 스토리 완료 후 Story 1.3의 검증/재시도 복구 강화가 동일 경로에서 검증 가능해야 한다.

### Previous Story Intelligence (Story 1.1)

- Story 1.1에서 서버 컨텍스트 기반 `ownerId` 정책으로 전환했으므로, Story 1.2도 요청 본문에 `ownerId`를 받지 않는다.
- API 계약은 이미 `{ data, meta }` / `{ error }` 형태로 고정되어 있으며, 신규 라우트도 동일 패턴을 따라야 한다.
- `x-trace-id` 정규화, `INVALID_JSON` 처리, 동일 화면 재시도 UX가 반영되어 있으므로 회귀 금지.
- `listingDraftStore`는 상한(500) 정책이 있으므로 조회/수정 테스트는 고정 fixture로 결정론적으로 수행한다.

### Architecture Compliance (Must Follow)

- 레이어 경계: `feature -> domain -> infra`만 허용, `feature -> infra` 직접 import 금지
- 매핑 경계: casing 변환은 `src/infra/mapper` 전용
- API 응답 규약: 성공 `{ data, meta }`, 실패 `{ error: { code, message, details?, requestId } }`
- Scope 우선순위: Active MVP 우선, Legacy Reference 기반 과구현 금지
- merge 순서: `contracts -> adapters/mapper -> handlers(route) -> UI`
