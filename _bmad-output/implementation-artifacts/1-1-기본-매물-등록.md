## Story

As a 판매 사용자,  
I want 익숙한 중고 매물 등록 흐름으로 물건을 저장하고 싶다,  
so that 설명 없이도 등록을 시작할 수 있다.

## Acceptance Criteria

1. 제목, 카테고리, 핵심 스펙 1개 이상, 가격을 입력하고 저장할 수 있다.
2. 저장 후 생성된 매물 상세를 조회할 수 있다.
3. 저장 실패 시 동일 화면에서 다시 시도할 수 있다.

### Story Intent

- Story 1.1의 목표는 "익숙한 매물 등록 저장 + 저장 직후 상세 조회 + 실패 시 같은 화면 재시도"까지다.

- UX 원칙은 "설명 없는 가치"와 "익숙한 폼 패턴 우선"이다. 새 UI를 실험적으로 만들지 말고, 일반적인 중고 매물 등록 폼을 닮아야 한다.
- 입력 폼은 모바일 우선 단일 컬럼 흐름이 기본이다.
- 오류는 필드와 연결되어야 하며, 저장 실패 메시지는 `aria-live="polite"` 또는 동등한 접근성 패턴으로 전달한다.

### Latest Technical Notes

- Next.js 공식 `use server` 문서는 Client Component에서 Server Function을 쓸 때 전용 파일에 `use server`를 두는 패턴을 권장하고, 민감한 서버 연산 전 인증/인가 확인을 먼저 하라고 안내한다. 현재 Story 1.1에서는 full auth를 도입하지 않더라도, 액션 설계는 이 패턴을 따라야 한다.
- Next.js 공식 Forms 가이드는 `useActionState`의 `pending` boolean으로 제출 버튼 비활성화와 loading indication을 구현할 수 있다고 설명한다. Story 1.1의 "같은 화면 재시도" UX에 잘 맞는다.
- React 공식 `useActionState` 문서는 알려진 오류를 action state로 반환하고, 예상 못 한 오류는 Error Boundary로 올리는 패턴을 설명한다. Story 1.1 저장 실패 UX도 이 구분을 따르는 편이 안전하다.
- MUI 공식 Next.js integration 문서는 App Router에서 `AppRouterCacheProvider` 사용을 권장하고, Next.js 16에서는 MUI `component` prop에 `next/link`를 직접 넘길 때 wrapper Client Component가 필요할 수 있다고 설명한다. 현재 저장소는 custom `ThemeRegistry`로 동등한 SSR style 수집을 수행하므로, 이유 없이 교체하지 말고 기존 패턴을 유지한다.
- Prisma 공식 Next.js 가이드는 dev 환경에서 PrismaClient singleton 패턴을 사용하고, Node.js `v20.19+`, `v22.12+`, `v24.0+`를 전제로 안내한다. 현재 저장소 엔진 범위와도 정합적이다.
